Observability contract.

Workspace operators can inspect latency, failure rate, slow routes, runtime cache, API volume, and ingestion history.

Latency

Workspace API events record route timing in metadata so slow searches and memory writes show up quickly.

Failure rate

The observability endpoint samples recent calls and reports failed routes without exposing private memory content.

Ingest health

Import runs stay visible beside runtime cache state, making large data loads easier to debug.

Runbook.

Concrete responses for common hosted-memory incidents.

noisy key
Inspect usage, create a replacement key with a clearer label, update the client, then revoke the noisy key.
bad ingest
Review ingest run metadata, export recent memories, forget bad records if needed, then re-import with smaller chunks.
stale context
Use search, recall hints, and session handoff snapshots to identify which state should be challenged or superseded.
client failure
Fetch `/connect`, verify the endpoint URL, list bridge tools, then test a small `recall_recent` call.
